# AGENTS.md

本文件定义 AI Agent / 自动化协作者在本仓库中的工作约束。执行任何代码、文档、测试或 PR 相关任务时，必须优先遵守本文件。

## 基本原则

- 先保护用户已有改动，再开始实现。
  - 开始修改前必须检查 `git status --short`。
  - 如果发现用户已有未提交改动，必须先说明并确认处理方式；可在用户同意后使用 stash 隔离。
- 不要静默删除已有能力、文档承诺或有用注释。
- 对外 API、README、包文档和示例必须保持一致。
- 每次创建 PR 或更新 PR 前，必须按 [`.agents/code-review.md`](./.agents/code-review.md) 执行自查 Code Review。

## 文档与 API 变更规则

- 删除或改写 README / docs 中已有 API 说明前，必须确认源码中是否仍存在对应能力。
- 如果删除文档内容是因为“文档与真实代码不一致”，需要在回复或 PR 描述中说明原因。
- 不允许为了让文档看起来简洁而删除仍有效的使用说明、限制说明或迁移提示。
- 新增配置、方法、事件、slot、ref/expose 能力时，必须同步更新相关 README / docs / 类型说明。

## 注释保留规则

- 未经用户允许，不得删除已有且有用的代码注释。
- 如果注释对应的代码已经失效、误导或为空逻辑，可以删除，但必须在回复或 PR 描述中说明原因。
- 重构代码时，如果原注释仍表达有效意图，应迁移或改写到新代码旁边。
- 删除调试日志不等同于删除注释；调试日志可以清理，但应说明理由。

## 测试规则

- 新增或修改单测时，`describe` / `it` 描述优先使用中文。
- 既有英文测试描述如非本次修改范围，可以不强制重写；如果本次新增测试文件，必须使用中文描述。
- DOM 尺寸相关测试应使用 mock，不依赖真实浏览器布局。
- 涉及 timer / requestAnimationFrame 的测试应避免 flaky，并显式清理 fake timers。

## PR 规则

- PR 前必须执行 [`.agents/code-review.md`](./.agents/code-review.md) 中的自查清单。
- PR 描述必须包含：
  - 变更摘要
  - 验证命令
  - 已知 warning 或未覆盖项
- 如果本地有用户 stash 或未合并改动，PR 描述或回复中必须提醒用户。
